# HC05

`HC05` 模块用于外接 HC-05/HC-06 一类串口蓝牙模块。它不是达妙 MC02 主控板的板载蓝牙驱动,当前工程默认也没有把它加入 `CMakeLists.txt`。

## 工作方式

- `HC05Init()` 接收一个 `UART_HandleTypeDef *`,并通过 `USARTRegister()` 注册到底层 `bsp/usart`。
- `bsp/usart` 负责启动 `ReceiveToIdle` 接收、维护接收队列、处理 DMA Cache 和发送忙状态。
- `HC05RxCallback()` 由 `USARTProcess()` 在 BSP 服务任务上下文中调用,不会在 UART 中断里直接解析协议。
- `HC05_SendData()` 组包后调用 `USARTSend()` 发送。
- `HC05_GetData()` 用于从模块内部拷贝最近一次收到的有效载荷,读取后会清空本次接收长度,避免上层重复处理同一帧。

## 协议格式

当前模块使用轻量串口帧:

```text
0xAA + len + payload[len] + checksum + 0x55
```

字段说明:

- `0xAA`: 帧头。
- `len`: 有效载荷长度,范围为 `1 ~ HC05_DATASIZE`。
- `payload`: 有效载荷数据。
- `checksum`: `len` 与所有 `payload` 字节逐字节异或的结果。
- `0x55`: 帧尾。

接收端按字节流状态机解析,可以处理半帧、粘包和一次串口回调内的多帧数据。若解析过程中发现长度、校验或帧尾错误,状态机会重新等待帧头;如果当前错误字节本身是 `0xAA`,会立即按下一帧帧头继续解析。

## 使用示例

```c
#include "HC05.h"
#include "usart.h"

static HC05 *bluetooth;

void BluetoothInit(void)
{
    bluetooth = HC05Init(&huart8);
}

void BluetoothSendExample(void)
{
    uint8_t data[HC05_DATASIZE] = {1, 2, 3, 4};

    if (bluetooth != NULL)
    {
        (void)HC05_SendData(data, sizeof(data));
    }
}

void BluetoothPollExample(void)
{
    uint8_t data[HC05_DATASIZE];
    uint8_t len;

    len = HC05_GetData(data, sizeof(data));
    if (len > 0U)
    {
        // 在这里处理收到的payload。
    }
}
```

## FreeRTOS 说明

`HC05` 的协议解析运行在 BSP 服务任务上下文中,符合当前工程“中断只收集事件,任务里处理”的结构。

接收数据保存在模块内部静态对象中。`HC05_GetData()` 使用短临界区拷贝数据并清空 `recv_len`,保证上层任务不会读到半帧更新状态。`HC05_SendData()` 也会先在短临界区保存最近一次发送帧,再调用 `USARTSend()`。

`USARTSend()` 在 IT/DMA 模式下会把发送数据复制到 USART 实例内部缓冲区,因此 `HC05_SendData()` 使用局部临时帧缓冲是安全的。

## CubeMX 与 BSP 依赖

若后续启用该模块,需要满足:

- 选择一条空闲 UART/USART 接口连接外接 HC-05 模块。
- 在 CubeMX 中配置对应 UART 的 TX/RX 引脚和波特率。HC-05 常见默认波特率为 `9600`,也可能被配置为其他值。
- 若使用 DMA 接收,需要开启对应 UART RX DMA 和 UART global interrupt。
- 若不使用 DMA,仍需要开启 UART global interrupt,以便 `ReceiveToIdle_IT` 工作。
- 工程中需要重新把 `modules/bluetooth/HC05.c` 和 `modules/bluetooth` include 路径加入 `CMakeLists.txt`。

## 注意事项

- 当前模块只支持一个 HC-05 实例。
- 当前帧载荷最大长度由 `HC05_DATASIZE` 决定,默认是 `4` 字节。
- `HC05_SendData()` 返回 `HAL_BUSY` 表示上一帧还没有发送完成,本次发送没有启动;上层需要自行决定重试或丢弃。
- `HC05_GetData()` 是读取即消费接口,如果上层没有及时读取,新的有效帧会覆盖旧帧。
- 该模块适合临时调参、简单遥控或少量状态数据传输,不适合作为高带宽可靠通信链路。
