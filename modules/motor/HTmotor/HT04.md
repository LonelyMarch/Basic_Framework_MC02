# HT04 电机驱动

## 适用范围

本驱动只适用于使用 MIT/Cheetah 风格 8 字节 CAN 协议的 HT04。它不适用于 `doc/HT/` 中使用命令码协议的 HTM35xx/HTM43xx。

HT04 内部完成 FOC 和力矩控制，主控侧不增加 PID。本模块固定采用直接前馈力矩控制：位置、速度、Kp、Kd 字段均打包为物理零值，只改变 12 bit 力矩字段。

## 任务与通信路径

HT04 不为每个物理电机创建任务。统一的 `MotorControlTask()` 约每 1 ms 调用一次 `HTMotorControl()`，后者遍历全部已注册实例并提交 CAN 控制帧。

```text
application
    └─ HTMotorSetTorque()
          只更新实例中的力矩目标

MotorControlTask()
    └─ HTMotorControl()
          ├─ 按方向配置处理力矩符号
          ├─ 完整打包 8 字节控制帧
          └─ CANTransmit()

FDCAN RX中断
    └─ BSP CAN接收事件队列
          └─ CANProcessTask
                └─ HTMotorDecode()
                      ├─ 校验DLC和Motor ID
                      ├─ 解析位置、速度、相电流
                      └─ DaemonReload()
```

CAN发送由FDCAN硬件异步完成。`CANTransmit()`成功表示报文已提交到硬件TX FIFO，不表示电机已经执行。

## 协议范围

控制帧固定为8字节：

| 字段 | 位宽 | 映射范围 |
|---|---:|---:|
| position | 16 bit | -95.5～95.5 rad |
| velocity | 12 bit | -45～45 rad/s |
| Kp | 12 bit | 0～500 N·m/rad |
| Kd | 12 bit | 0～5 N·m·s/rad |
| feed-forward torque | 12 bit | -18～18 N·m |

反馈帧固定为6字节：

| 字段 | 位宽 | 映射范围 |
|---|---:|---:|
| Motor ID | 8 bit | 电机固件配置值 |
| position | 16 bit | -95.5～95.5 rad |
| velocity | 12 bit | -45～45 rad/s |
| phase current | 12 bit | -40～40 A |

注意：发送字段是力矩，反馈字段是峰值相电流，二者范围和单位不同。

特殊命令的前7字节均为 `0xFF`：

| DATA[7] | 含义 |
|---:|---|
| `0xFC` | 进入 Motor Mode |
| `0xFD` | 退出 Motor Mode |
| `0xFE` | 将当前位置设置为机械零点 |

## 注册实例

```c
HTMotor_Init_Config_s config = {
    .can_init_config = {
        .can_handle = &hfdcan1,
        .tx_id = 1,
        .rx_id = 1,
    },
    .motor_reverse_flag = MOTOR_DIRECTION_NORMAL,
    .expected_motor_id = 1,
    .enable_on_init = 0,
    .set_zero_on_init = 0,
};

HTMotorInstance *motor = HTMotorInit(&config);
```

- `expected_motor_id = 0` 表示不检查反馈 DATA[0]；通常应填写真实ID。
- 默认不建议在初始化时自动使能。
- 默认不建议在初始化时自动设置零点，避免每次上电改变机械坐标系。
- 控制模式固定为直接力矩通信，没有运行期模式切换接口。

## 控制接口

```c
HAL_StatusTypeDef HTMotorEnable(HTMotorInstance *motor);
HAL_StatusTypeDef HTMotorSetTorque(HTMotorInstance *motor, float torque_nm);
HAL_StatusTypeDef HTMotorStop(HTMotorInstance *motor);
HAL_StatusTypeDef HTMotorDisable(HTMotorInstance *motor);
HAL_StatusTypeDef HTMotorSetCurrentPositionAsZero(HTMotorInstance *motor);
```

- `HTMotorEnable()`发送 `0xFC`，提交成功后进入软件使能状态。
- `HTMotorSetTorque()`只在在线、已使能且未停止时接受目标。
- `HTMotorStop()`清零目标并持续发送零力矩，但保持 Motor Mode。
- `HTMotorDisable()`清零目标并发送 `0xFD`，退出 Motor Mode。
- `HTMotorSetCurrentPositionAsZero()`发送 `0xFE`；该操作不会在初始化时强制执行。

## 离线安全行为

合法反馈会重载daemon。反馈超时后模块只记录一次离线日志，并立即：

- 清零力矩目标；
- 锁定停止状态；
- 清除软件使能状态；
- 禁止自动带旧目标重新使能。

反馈恢复时只记录一次恢复日志，电机仍保持停止。上层必须重新调用 `HTMotorEnable()`，然后下发新的力矩目标。

## 方向语义

`motor_reverse_flag`作用于命令和反馈：

- 发送力矩反向；
- 位置反馈反向；
- 速度反馈反向；
- 相电流符号反向。

这样上层看到的是统一后的逻辑坐标系。
