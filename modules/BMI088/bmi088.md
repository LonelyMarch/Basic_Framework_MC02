# BMI088

`modules/BMI088` 负责板载 BMI088 的完整使用链路：`bmi088.c/.h` 处理 SPI 通信、寄存器初始化、数据解码、DRDY 事件和标定参数，
`ins_task.c/.h` 处理温控 PID、加热 PWM、安装误差修正和四元数 EKF 姿态解算。原 `modules/imu` 的板载惯导职责已经合并到本目录。

外接达妙 DM-IMU-L1 不属于本模块，由独立的 `modules/DMimu` 通过 CAN 或 RS485 通信获取已经由外部 IMU 解算好的数据。

当前工程只面向达妙 MC02 主控板上的单个 BMI088，驱动内部使用静态单实例，重复调用 `BMI088Register()` 会返回同一个实例。

## 硬件连接

BMI088 的加速度计和陀螺仪共用 `SPI2`，但使用两根片选线：

- 加速度计片选：`CS2_ACCEL`
- 陀螺仪片选：`CS2_GYRO`
- 加速度计 DRDY：`ACC_INT`
- 陀螺仪 DRDY：`GYRO_INT`

SPI 访问通过 `bsp/spi` 完成。`BMI088Register()` 会把加速度计和陀螺仪分别注册成两个 `SPIInstance`，片选控制、总线互斥和 HAL
返回值处理由 BSP SPI 统一负责。

## 传感器配置

当前初始化寄存器配置为：

- 加速度计量程：`6G`
- 加速度计输出数据率：`1600Hz`
- 陀螺仪量程：`2000dps`
- 陀螺仪输出数据率/带宽：`2000Hz / 230Hz`
- 温度：从加速度计侧温度寄存器读取

SPI 读写规则与 BMI088 数据手册、达妙官方例程保持一致：

- 加速度计读寄存器时，地址字节后有 1 个额外 dummy byte，有效数据从接收缓冲区第 3 个字节开始。
- 陀螺仪读寄存器时，地址字节后立即返回有效数据，有效数据从接收缓冲区第 2 个字节开始。
- 温度原始值为 11 位有符号数，解码公式为 `raw * 0.125f + 23.0f`。

## 工作模式

### 周期模式

`BMI088_BLOCK_PERIODIC_MODE` 是当前默认主路径。上层任务周期调用 `BMI088Acquire()`，驱动主动读取一次加速度计和陀螺仪数据。

周期模式下，BMI088 不需要输出 DRDY 外部中断：

- 加速度计只关闭 `INT_MAP_DATA`，保留 `INT1_IO_CTRL` 的确定输出配置，避免 MCU 侧 `ACC_INT` 输入悬空。
- 陀螺仪关闭 `GYRO_CTRL` 和 `GYRO_INT3_INT4_IO_MAP`，避免周期读取时产生无意义 EXTI。

温度变化远慢于姿态数据。驱动内部用 `BMI088_TEMP_UPDATE_DIVIDER` 对温度读取降频，当前约每 `1000` 次采样路径读取一次温度，按
1kHz INS 任务折算约 `1Hz`。其余周期沿用最近一次有效温度。

### 触发模式

`BMI088_BLOCK_TRIGGER_MODE` 用于 DRDY 触发读取。只有该模式会配置 BMI088 的 DRDY 输出和中断映射，并注册 `ACC_INT/GYRO_INT`
的 GPIO 回调。

触发模式的关键原则是：中断只记录事件，SPI 读取仍在任务上下文中完成。EXTI 回调不会直接访问 SPI，而是更新加速度计和陀螺仪的
DRDY 计数。`BMI088Acquire()` 会在任务中检查两路计数，只有 acc 和 gyro 都有未处理 DRDY 时，才读取并发布一帧完整 IMU 数据。

为了减少 acc/gyro 跨采样窗口混用，触发模式会在 SPI 读取前后各快照一次 DRDY 计数：

- 如果读取前 acc 或 gyro 任一路没有新事件，函数返回 `0`，等待另一半数据。
- 如果读取过程中又来了新的 DRDY，说明寄存器可能已经被下一次采样刷新，驱动会重读一次。
- 如果连续重试仍不稳定，驱动会丢弃已观察到的事件，等待下一对 DRDY。

这个逻辑保证软件层尽量发布同一采样窗口的数据，但它不是硬件级严格同步采样。

启用触发模式时，需要在 CubeMX 中开启 `EXTI line[15:10] interrupts`，当前建议优先级为 `5,0`。模块只注册 GPIO 回调，不在
BMI088 驱动内配置 NVIC。

## 初始化接口

```c
BMI088Instance *BMI088Register(BMI088_Init_Config_s *config);
```

`BMI088Register()` 注册并初始化 BMI088。成功返回实例指针，失败返回 `NULL`。

初始化流程为：

1. 检查配置参数。
2. 注册加速度计和陀螺仪两个 SPI 从设备实例。
3. 读取两次加速度计 `CHIP_ID`，让芯片稳定进入 SPI 访问路径。
4. 对加速度计执行软复位、基础寄存器配置和模式寄存器配置。
5. 对陀螺仪执行软复位、基础寄存器配置和模式寄存器配置。
6. 初始化失败时最多重试 `BMI088_INIT_RETRY_MAX` 次。
7. 按 `cali_mode` 执行在线标定或加载离线参数。
8. 若工作模式为触发模式，注册 `ACC_INT/GYRO_INT` 的 GPIO 回调。

当前初始化发生在 `RobotInit()` 阶段，FreeRTOS 尚未开始调度，工程全局中断也可能处于关闭状态。因此驱动会强制 SPI
使用阻塞模式完成初始化、寄存器校验和标定。进入任务后仍通过同步接口读取，由 BSP SPI 负责总线互斥。

典型配置由同目录的 `ins_task.c` 生成：

```c
BMI088_Init_Config_s bmi088_config = {
    .work_mode = BMI088_BLOCK_PERIODIC_MODE,
    .cali_mode = BMI088_CALIBRATE_ONLINE_MODE,
    .spi_gyro_config = {
        .spi_handle = &hspi2,
        .GPIOx = CS2_GYRO_GPIO_Port,
        .cs_pin = CS2_GYRO_Pin,
        .spi_work_mode = SPI_BLOCK_MODE,
    },
    .spi_acc_config = {
        .spi_handle = &hspi2,
        .GPIOx = CS2_ACCEL_GPIO_Port,
        .cs_pin = CS2_ACCEL_Pin,
        .spi_work_mode = SPI_BLOCK_MODE,
    },
    .gyro_int_config = {
        .GPIOx = GYRO_INT_GPIO_Port,
        .GPIO_Pin = GYRO_INT_Pin,
        .pin_state = GPIO_PIN_RESET,
        .exti_mode = GPIO_EXTI_MODE_RISING,
    },
    .acc_int_config = {
        .GPIOx = ACC_INT_GPIO_Port,
        .GPIO_Pin = ACC_INT_Pin,
        .pin_state = GPIO_PIN_RESET,
        .exti_mode = GPIO_EXTI_MODE_RISING,
    },
};
```

`work_mode` 是初始化时决定 BMI088 硬件 DRDY 配置的模式。代码中初始化采样或标定时临时改写 `bmi088->work_mode`，只是在软件上选择
`BMI088Acquire()` 的读取分支，不会重新配置传感器 DRDY 寄存器，因此不应把它当成完整的运行期硬件模式切换接口。

## 采样接口

```c
uint8_t BMI088Acquire(BMI088Instance *bmi088, BMI088_Data_t *data_store);
```

`BMI088Acquire()` 读取一次 BMI088 数据。返回 `1` 表示本次获得新的有效 IMU 数据，返回 `0` 表示读取失败，或触发模式下完整
acc+gyro 帧尚未到齐。

输出单位为：

- `data_store->gyro[]`：`rad/s`
- `data_store->acc[]`：`m/s^2`
- `data_store->temperature`：摄氏度

周期模式下，函数按顺序读取加速度、陀螺仪和降频温度。加速度或陀螺仪读取失败会返回 `0`；温度读取失败只会限频打印日志，并沿用上一帧温度，不阻断姿态数据更新。

触发模式下，函数以完整 acc+gyro 帧为边界。若只有一路 DRDY 到达，不读取、不清标志，直接返回 `0`。上层可据此跳过本周期 EKF
更新，而不是把半帧数据当作新姿态输入。

## 标定接口

```c
void BMI088CalibrateIMU(BMI088Instance *bmi088);
```

`BMI088Register()` 内部已经会根据 `cali_mode` 调用该函数，上层通常不需要再次主动调用。标定是阻塞式流程，应只在系统初始化阶段执行。

标定结果写入：

- `gyro_offset[]`：三轴陀螺仪零偏，单位 `rad/s`
- `gNorm`：当前板子静止时测得的重力模长
- `acc_coef`：加速度计原始值到 `m/s^2` 的换算系数
- `temp_when_cali`：标定时温度，为后续温漂补偿预留

### 在线标定

`BMI088_CALIBRATE_ONLINE_MODE` 会在上电初始化时执行在线标定：

1. 保存当前工作模式，临时切到周期读取分支。
2. 清空旧零偏，把 `acc_coef` 恢复为当前量程基础灵敏度。
3. 连续采样 `6000` 次，每次间隔 `0.0005s`。
4. 累加陀螺仪三轴数据，计算静止零偏。
5. 计算每帧加速度模长，估计当前重力模长。
6. 记录采样窗口内的最大/最小值，判断标定期间是否发生晃动或转动。
7. 若采样不完整、波动过大或候选结果不合理，丢弃本轮候选值并重试。
8. 若总耗时超过 `12s`，切换到离线参数。

候选结果通过检查后才会写入实例，失败候选不会污染下一轮标定。在线标定要求整车或云台保持静止。

### 离线参数

`BMI088_LOAD_PRE_CALI_MODE` 会直接加载代码中的预设离线参数。在线标定超时或失败时也会自动回退到离线参数。

离线参数由 `BMI088_OFFLINE_PARAM_ID` 选择：

- 若工程定义了 `INFANTRY_ID`，则使用 `INFANTRY_ID`。
- 若未定义 `INFANTRY_ID`，默认使用 `0` 号参数。

离线参数适合追求快速上电，但需要确保参数来自当前这块板子的实际标定结果。

## FreeRTOS 约定

- 初始化和标定在任务启动前完成，不依赖 RTOS 调度。
- 运行期不要在 EXTI 中断里直接读 BMI088，所有 SPI 事务都由任务上下文调用 `BMI088Acquire()` 完成。
- 触发模式下 `BMI088Acquire()` 返回 `0` 不一定表示硬件故障，也可能只是完整帧尚未到齐。
- SPI 异常日志使用限频输出，避免 1kHz 姿态任务在故障时持续刷屏。
- 当前驱动为单实例设计，适配板载 BMI088；不建议在同一工程中再为另一个 BMI088 复用该静态实例。
